# Plan: Complete Thumbnaily Deployment & Enhancement with OpenAI Responses API

## Context

Thumbnaily (Next.js + Supabase + Netlify) is being enhanced to leverage the **OpenAI Responses API** as the primary orchestration layer for AI-powered thumbnail generation and broader creative asset production. The Responses API combines image generation, vision analysis, tool calling, conversation memory, and multi-modal orchestration in a single interface.

## Architecture: Platform-Wide Responses API Integration

### Core Engine Layer
All Thumbnaily apps (VideoRemix, AI Agency, Marketing Studio, Thumbnail Generator) share a single image generation engine:

```
OpenAI Responses API
        │
        ▼
Image Generation Engine
        │
 ├── Prompt Builder (structured prompt assembly)
 ├── Brand System (consistent styling across apps)
 ├── Template Engine (variable replacement, editable regions)
 ├── Style Presets (lighting, mood, quality presets)
 ├── Asset Library (reference images, logos, backgrounds)
 ├── Reference Image Manager (multi-image composition)
 ├── Batch Generation (parallel variant creation)
 ├── Version History (multi-turn conversation state)
 ├── Multi-step Editor (conversational editing)
 └── Export Manager (size, format, quality controls)
```

### Thumbnaily-specific Implementation Map

| Capability | Responses API Feature | Thumbnaily Use Case |
|------------|----------------------|---------------------|
| Text → Image | `image_generation` tool | Primary thumbnail generation |
| Image + Prompt | Input images + text | Reference assets + custom prompt |
| Vision Analysis | `vision` tool | Analyze current thumbnails, extract text, check layout |
| Multi-turn Editing | `previous_response_id` | Iterative refinement: "remove shadow → make brighter → white bg" |
| Conversation Memory | Thread state | Undo/redo, version history |
| Variants | `n: 4` in batch mode | 4-variant toggle for A/B testing |
| Caption Overlay | Edit mode with text overlay asset | YouTube title + thumbnail |
| Background Replace | `action: "edit"` with mask | Change background while preserving subject |
| Multi-Channel | Multi-turn with aspect ratio changes | Resize for YouTube (1280×720), Shorts (1080×1920), Instagram (1080×1080) |
| Web Search + Images | `web_search` tool | Research trending thumbnails, then generate |
| File Search + Images | `file_search` tool | Read brand guide PDF, apply to thumbnail |
| Streaming | `stream: true` | Real-time partial image previews |
| Quality Controls | `quality`, `format`, `background` | HD, transparent PNG, etc. |

### Template Engine & Library (Next.js + Supabase)

Thumbnaily implements a custom template engine layered on top of the Responses API. Templates are stored in Supabase (`prompt_templates` + `style_presets` tables), instantiated client-side via `/api/templates`, and executed through `/api/generate-thumbnail` with variable interpolation.

#### Template Data Model

```json
{
  "id": "string (slug)",
  "category": "string",
  "subcategory": "string",
  "name": "string (display name)",
  "description": "string",
  "prompt": "string (base prompt with {{variables}})",
  "variables": [
    { "name": "product_name", "label": "Product Name", "required": true, "maxLength": 100 }
  ],
  "style": "string (references style_presets.slug)",
  "dimensions": { "width": 1280, "height": 720 },
  "quality": "high",
  "background": "auto",
  "referenceImages": ["url or null"],
  "editableRegions": [{ "bbox": [x,y,w,h], "label": "text area" }],
  "systemPrompt": "string (optional)",
  "tools": ["image_generation", "vision"],
  "mode": "default | variants | caption | background | channel",
  "n": 1,
  "tags": ["trending", "ctr-optimized"]
}
```

#### Template Engine Implementation

Every template in the database is executable through a unified pipeline:

1. **Interpolation**: `prompt.replace(/{{(\w+)}}/g, (_, key) => variables[key])`
2. **Responses API Call**: POST `/api/generate-thumbnail` with body:
   ```json
   {
     "prompt": "<interpolated>",
     "mode": "<template.mode>",
     "n": "<template.n>",
     "quality": "<template.quality>",
     "dimensions": "<template.dimensions>",
     "background": "<template.background>",
     "referenceImages": "<template.referenceImages>",
     "style": "<template.style>",
     "systemPrompt": "<template.systemPrompt>",
     "tools": "<template.tools>"
   }
   ```
3. **Streaming Response**: Same SSE channel used for basic generation; emits `partial-variant` events.
4. **Post-Processing**: If `editableRegions` are defined, return bounding boxes for text overlay; if `mode: "caption"`, auto-attach YouTube title/default caption.

#### UI Integration

- `/app` page gains a **Template Gallery** tab alongside existing Prompt Coach / Style / History panels. Filters: category → subcategory → mode.
- Selected template pre-fills the prompt builder, drops in predefined variables, and sets channel/quality defaults.
- Template variants for A/B testing: user selects Batch (n=4) and Thumbnaily generates all variants in one Responses API call.
- Conversation tree auto-associates generated images with the originating template ID for traceability.

#### How This Maps to OpenAI Responses API

- Text → Image: every template ultimately calls `image_generation` tool.
- Image + Prompt: templates declaring `referenceImages` send those as base64/file_url alongside text.
- Vision Analysis: if `tools` includes `"vision"`, the engine sends the reference image through the `vision` tool before generation to extract composition notes.
- Multi-turn: selecting a generated variant and applying "Background Replace" chains a new Responses API call with `previous_response_id` so the conversation memory persists.
- Web Search + Images: templates in "Trending" or "News" subcategories enable `tool: "web_search"` mode; engine summarizes search then generates.
- File Search + Images: templates with `referenceImages` pointing to PDF/Doc URLs trigger `file_search` first.
